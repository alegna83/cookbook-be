# Usability Study — Stays4Pilgrims

Resumo rápido: este diretório contém templates e instruções para conduzir um estudo de usabilidade assíncrono para a aplicação, em contexto académico. As sessões são destinadas a avaliar a usabilidade do protótipo — não há compensação financeira.

- Passos principais:
- Definir objetivos e KPIs (task success, time-on-task, SUS, qualitative feedback)
- Preparar materiais: roteiro do moderador, consentimento, questionário SUS, mensagem de recrutamento
- Realizar piloto com 2–3 participantes e ajustar o roteiro
- Conduzir sessões (30–45 min cada)
- Compilar e analisar métricas quantitativas e qualitativas

Arquivos:
- `script.md` — roteiro + tarefas e perguntas
- `consent.md` — formulário de consentimento
- `recruitment.md` — mensagem de recrutamento
- `sus.md` — questionário SUS

Ferramentas recomendadas: Jitsi/Zoom (record), Lookback/Maze/Hotjar (fluxo), Google Forms (questionários), OBS (local recording)
